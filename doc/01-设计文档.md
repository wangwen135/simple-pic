# Simple-Pic 本地图床应用设计文档

## 项目概述

开发一个功能完整的本地图片上传应用，基于 Spring Boot + Java 1.8。支持多用户、多存储空间、后台管理、API上传、缩略图生成、水印等功能，零数据库依赖。

---

## 技术选型

- **后端**: Spring Boot 2.7.x (兼容 Java 1.8)
- **前端**: 纯 HTML + 原生 JavaScript + Tailwind CSS（科技感现代配色）
- **存储**: 本地文件系统 + YAML 配置文件
- **缓存**: 内存缓存（登录会话、IP限流）
- **图片处理**: Java AWT / Thumbnailator

## 核心功能模块

### 1. 认证与权限模块

**功能说明**:
- 登录页面（背景图片 + 悬浮登录框）
- 用户名密码存储在 config.yml
- 角色管理：普通用户 / 管理员
- 记住我功能：内存存储会话，30天有效期
- 权限控制：普通用户仅上传，管理员可管理

**技术实现**:
- 使用 BCrypt 加密存储密码
- 使用 ConcurrentHashMap 存储登录会话
- 会话结构：`{username, loginTime, expiryTime, storageSpace}`
- 定时任务清理过期会话

### 2. 图片上传模块

**功能说明**:
- 支持三种方式：文件选择、拖拽、粘贴
- 支持格式：jpg, png, gif, webp, svg
- 存储路径：`{当前存储空间}/{4位年份}/{2位月份}/{UUID}.{后缀}`
- 生成公开 URL（域名在后台配置）
- 多格式链接复制：
  - Markdown（默认，自动复制）
  - 直链
  - HTML `<img>` 标签
  - BBCode
- 右上角显示：当前存储空间、当前登录用户

**技术实现**:
- 使用 Spring MultipartFile 处理文件上传
- 使用 UUID 生成唯一文件名
- 支持文件拖拽和剪贴板粘贴（前端 JavaScript）

### 3. 存储空间模块

**功能说明**:
- 多存储空间支持，可切换
- 每个存储空间配置：目录路径、大小限制
- 超过大小限制禁止上传并提示
- 异步任务检查存储空间使用情况

**技术实现**:
- 使用 File API 获取文件大小和修改时间
- 缓存存储统计信息（5分钟TTL）
- 支持的单位：KB, MB, GB

### 4. 后台管理模块（管理员专属）

**仪表盘**:
- 存储空间统计（图片数量、总大小、子目录数等）
- 活跃会话数
- 系统信息

**图片管理**:
- 按目录层级查看
- 查看已上传图片，删除图片
- 后台上传图片
- 目录操作（新建、删除）

**存储空间配置**:
- 增删改存储空间配置
- 查看存储空间使用情况

**用户配置**:
- 创建用户，设置密码、角色、关联存储空间
- 管理现有用户

**系统设置**:
- 基础信息（系统名称、描述）
- 前端配置（主题、显示参数）
- 水印设置（文字/图片、位置、透明度）
- 安全设置（IP 限流配置）

**API Key 配置**:
- 管理 API 上传令牌

### 5. 缩略图模块

**功能说明**:
- 上传时自动生成缩略图
- 后台定时任务扫描并生成缺失的缩略图
- 缩略图不存在时显示原图

**技术实现**:
- 使用 Thumbnailator 库生成缩略图
- 缩略图尺寸：300x300
- 缩略图存放在 `.thumbnails/` 子目录
- 定时任务：每天凌晨3点执行

### 6. API 模块

**功能说明**:
- API 上传接口，使用 Token 鉴权
- Token 存储在 config.yml

**技术实现**:
- 支持 Bearer Token 和查询参数两种方式
- 健康检查接口

### 7. 日志模块

**功能说明**:
- 日志配置在 application.yml
- 自动归档压缩
- 最多保留 30 天

**技术实现**:
- 使用 Logback
- 单文件最大 100MB
- GZIP 压缩归档

## 配置文件结构 (config.yml)

```yaml
simple-pic:
  # 系统基础信息
  system:
    name: Simple-Pic
    description: 简单好用的本地图床

  # 存储空间配置
  storage-spaces:
    - name: default
      path: ./storage/default
      max-size: 10GB
      domain: http://localhost:8080

  # 用户配置
  users:
    - username: admin
      password: {BCrypt加密后的密码}
      role: ADMIN
      storage-spaces: [default]
    - username: user1
      password: {BCrypt加密后的密码}
      role: USER
      storage-spaces: [default]

  # API Keys
  api-keys:
    - token: xxxxx
      storage-space: default

  # 水印配置
  watermark:
    enabled: true
    type: text  # text 或 image
    content: Simple-Pic
    position: bottom-right
    opacity: 0.5

  # 安全配置
  security:
    rate-limit:
      enabled: true
      max-requests: 100
      time-window: 60  # 秒

  # 前端配置
  frontend:
    theme: dark
    items-per-page: 50
```

## 首次启动流程

1. 检测 config.yml 是否存在
2. 不存在则生成默认配置
3. 为 admin 账号生成随机密码，输出到控制台
4. 提示用户使用生成的密码登录后台进行配置

## 关键技术点

### 1. 无数据库的元数据存储

- 使用文件系统本身的目录结构和文件信息
- 图片上传时间从文件修改时间获取
- 文件大小通过 File API 获取
- 缩略图存放在 `.thumbnails/` 子目录

### 2. 会话管理

- 使用 ConcurrentHashMap 存储登录会话
- Session 结构：`{username, loginTime, expiryTime, storageSpace}`
- 定时任务清理过期会话

### 3. IP 限流

- 使用 Guava RateLimiter
- Map 存储每个 IP 的限流器
- 定时任务清理过期 IP 记录

### 4. 配置文件热更新

- 使用 SnakeYAML 读写 YAML
- 配置修改后立即生效
- 管理员可在后台修改配置

### 5. 图片处理

- Thumbnailator 生成缩略图
- AWT 添加文字水印
- Graphics2D 添加图片水印

## 前端 UI 设计要点

- 使用深色主题，科技感配色（深蓝/灰背景 + 蓝色/紫色强调色）
- 登录页：背景图片 + 半透明玻璃效果登录框
- 上传页：简洁的拖拽区域，右上角信息栏
- 管理后台：左侧导航栏 + 右侧内容区，响应式布局
- 所有交互有即时反馈（加载动画、成功/失败提示）
- 使用 Tailwind CSS CDN 实现现代样式
