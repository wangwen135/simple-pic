存储挂载方式说明

Simple-Pic 的存储空间支持多种挂载方式。本页面介绍如何通过不同的 Web 服务器将存储目录映射为可通过 URL 访问的资源。

工作原理

当你上传图片后，系统会根据存储空间的 访问路径前缀 配置来生成图片的访问 URL：

完整 URL = 访问路径前缀 + 文件相对路径
1

访问路径前缀

你配置的 URL 前缀，如 http://example.com/images/note/

2

文件相对路径

系统自动生成：年份/月份/UUID.扩展名，如 2026/04/a1b2c3d4.jpg

系统自带的图片 API 路径格式为：/image/{存储空间名称}/{文件相对路径}

1 使用系统内置 API（默认）

最简单的方式，无需额外配置。直接使用系统自带的图片访问接口。

访问路径前缀填写示例：

http://192.168.31.88:8080/images/mystorage/

系统会自动将 /images/{存储空间名}/ 映射到对应的存储目录
推荐方式，无需额外配置，开箱即用

2 Nginx 反向代理挂载

通过 Nginx 直接映射存储目录到 URL 路径，实现静态文件访问，性能更好。

基础配置：

复制 
server {
    listen 80;
    server_name example.com;

    # Simple-Pic 应用
    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 静态图片直接访问（性能更优）
    location /images/mystorage/ {
        alias /path/to/storage/;

        # 可选：设置浏览器缓存
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}

访问路径前缀填写：http://example.com/images/mystorage/

进阶配置（含缓存和跨域）：

复制 
# 静态图片访问 - 进阶配置
location /images/mystorage/ {
    alias /path/to/storage/;

    # 浏览器缓存
    expires 30d;
    add_header Cache-Control "public, immutable";

    # 跨域支持（如需在其他网站引用图片）
    add_header Access-Control-Allow-Origin *;
    add_header Access-Control-Allow-Methods "GET, OPTIONS";

    # 安全：禁止访问隐藏文件
    location ~ /\. { deny all; }
}
修改 Nginx 配置后需执行 nginx -s reload 重新加载

3 Caddy 反向代理挂载

如果你使用 Caddy 作为 Web 服务器，可以使用以下配置：

复制 
example.com {
    # Simple-Pic 应用
    reverse_proxy localhost:8080

    # 静态图片直接访问
    handle_path /images/mystorage/* {
        root * /path/to/storage
        file_server
        header Cache-Control "public, immutable"
    }
}

访问路径前缀填写：http://example.com/images/mystorage/

4 CDN 加速

将存储目录通过 CDN 分发，适用于需要全球加速访问的场景。

1 将存储目录配置为 CDN 的源站
2 在 CDN 控制台配置回源路径为存储目录
3 将访问路径前缀设置为 CDN 分发域名 
访问路径前缀填写示例：

https://cdn.example.com/images/mystorage/

常见问题

Q: 访问路径前缀末尾需要加 / 吗？

是的，建议始终以 / 结尾，确保 URL 拼接正确。

Q: 可以使用 HTTPS 吗？

可以。在 Nginx/Caddy 中配置 SSL 证书后，访问路径前缀使用 https:// 即可。

Q: 修改配置后图片无法访问？

检查：1) 存储目录路径是否正确；2) Web 服务器是否有读取权限；3) Nginx 配置是否已 reload；4) 访问路径前缀是否与 Nginx location 匹配。
